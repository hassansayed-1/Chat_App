1. Technology Stack & Architecture

The application is built on a modern, decoupled architecture designed to provide real-time communication while ensuring a Zero-Knowledge Server Environment. The server routes packets but possesses zero capability to read the plaintext.

1.1. Frontend Application

Next.js 15 React 19 TailwindCSS v4

The frontend is built with React using the Next.js App Router. We heavily utilize client-side rendering ('use client') specifically for the ChatContext. Why? By keeping the React Context entirely on the client, we ensure that cryptographic key generation (RSA, Kyber) and storage (via localStorage) strictly occur inside the user's browser sandbox. Private keys never traverse the network.

1.2. Networking & Backend

Node.js Express Socket.io

The backend operates as a pure signaling and relay server. Socket.io was chosen for its robust WebSocket support, enabling low-latency, bidirectional event-driven communication (essential for real-time chat). It manages rooms natively, allowing efficient broadcasting of encrypted payloads to specific groups of connected clients.

1.3. Cryptography Libraries

crypto-js node-forge crystals-kyber

  • crypto-js: Chosen for its mature, widely vetted implementation of symmetric algorithms (AES, DES, RC4).
  • node-forge: Provides robust pure-JavaScript implementations of Public Key Infrastructure (PKI), specifically RSA-OAEP and AES-GCM (which requires complex Galois field operations).
  • crystals-kyber: The bleeding-edge Post-Quantum Cryptography (PQC) library implementing the NIST ML-KEM standard to protect against future quantum computing attacks.

2. End-to-End Secure Messaging Flow

How does a message travel from Sender (Alice) to Receiver (Bob) securely over an untrusted Socket.io connection? The application implements a Hybrid Cryptosystem (combining Asymmetric Key Exchange with Symmetric Payload Encryption).

Step 1: Initialization & Key Generation

When Alice and Bob load the app, ChatContext.tsx checks localStorage for existing keys. If none exist, it generates new RSA-2048 and Kyber768 keypairs client-side.

Step 2: Public Key Distribution

Upon joining a room, Alice emits a share-public-key socket event. The server broadcasts her Public RSA and Public Kyber keys to everyone in the room (including Bob). Bob stores Alice's public keys in his React state.

Step 3: Message Encryption (Alice)

Alice types a message. A random, ephemeral symmetric Session Key (e.g., an AES key) is generated.

  • Payload: Alice encrypts the plaintext message with AES using this Session Key.
  • Encapsulation: Alice loops through every participant in the room. She takes the Session Key and encrypts it specifically for Bob using Bob's Public Key (via RSA or Kyber).

Step 4: Transmission

Alice sends a JSON payload to the server containing the single encrypted message, and a dictionary of encrypted session keys mapped by User ID.

Step 5: Decryption (Bob)

Bob receives the payload. He looks up his User ID in the dictionary, uses his Private Key to decrypt the Session Key, and finally uses that Session Key to decrypt the AES payload.

3. Classical Cryptography (Educational Mode)

To demonstrate cryptographic evolution, the app includes foundational ciphers.

3.1. Caesar Cipher (Affine Shift)

Mathematical Concept: Operations occur on the finite cyclic group \(\mathbb{Z}_{26}\).

\( E_k(x) = (x + k) \pmod{26} \)

\( D_k(x) = (x - k) \pmod{26} \)

Implementation:

export const caesarEncrypt = (text: string, shiftStr: string): string => {
  const shift = parseInt(shiftStr, 10) || 0;
  return text.split('').map(char => {
    const code = char.charCodeAt(0);
    if (code >= 65 && code <= 90) return String.fromCharCode(((code - 65 + shift) % 26 + 26) % 26 + 65);
    if (code >= 97 && code <= 122) return String.fromCharCode(((code - 97 + shift) % 26 + 26) % 26 + 97);
    return char;
  }).join('');
};

4. Symmetric Cryptography (Payload Encryption)

4.1. Advanced Encryption Standard (AES) Mathematical Proof

AES is a Substitution-Permutation Network operating on a 128-bit block size. Unlike classical modular arithmetic, AES operates entirely within the Galois Field \(GF(2^8)\) defined by the irreducible polynomial:

\( P(x) = x^8 + x^4 + x^3 + x + 1 \)

SubBytes & MixColumns

The SubBytes step provides "Confusion" by replacing a byte \(x\) with its multiplicative inverse \(x^{-1}\) in \(GF(2^8)\), followed by an affine transformation over \(GF(2)\).

The MixColumns step provides "Diffusion". Each column is multiplied modulo \(x^4 + 1\) with a fixed polynomial \(a(x) = \{03\}x^3 + \{01\}x^2 + \{01\}x + \{02\}\).

export const aesEncrypt = (text: string, key: string, mode: string = 'CBC'): string => {
  return CryptoJS.AES.encrypt(text, key, { mode: getCryptoJSMode(mode) }).toString();
};

4.2. Authenticated Encryption: AES-GCM (Galois/Counter Mode)

The application implements AES-GCM, which provides both Confidentiality and Authenticity (AEAD), preventing Chosen-Ciphertext Attacks (CCA).

Confidentiality via CTR Mode: AES encrypts a 96-bit Initialization Vector (IV) and a counter to generate a keystream, which is XORed (\(\oplus\)) with the plaintext \(P\).

\( C_i = P_i \oplus AES_K(IV \parallel Counter_i) \)

Authenticity via GHASH: GCM computes a Message Authentication Code (MAC) using polynomial multiplication over \(GF(2^{128})\) evaluated at the authentication subkey point \(H = AES_K(0^{128})\).

\( X_i = (X_{i-1} \oplus C_i) \cdot H \pmod{P(x)} \)

export const aesGcmEncrypt = (text: string, key: string) => {
  const iv = forge.random.getBytesSync(12);
  const cipher = forge.cipher.createCipher('AES-GCM', key);
  cipher.start({ iv, tagLength: 128 });
  cipher.update(forge.util.createBuffer(text, 'utf8'));
  cipher.finish();
  
  return {
    ciphertext: forge.util.encode64(cipher.output.getBytes()),
    iv: forge.util.encode64(iv),
    tag: forge.util.encode64(cipher.mode.tag.getBytes()) // The GHASH Tag
  };
};

5. Asymmetric Cryptography (Secure Key Exchange)

5.1. RSA-OAEP (Optimal Asymmetric Encryption Padding)

RSA's security relies on the Integer Factorization Problem. We compute modulus \(n = p \times q\) and totient \(\phi(n) = (p-1)(q-1)\). The keys are \(e\) (public) and \(d\) (private) where \(ed \equiv 1 \pmod{\phi(n)}\).

By Euler's Theorem, decryption works because:

\( C^d \equiv (M^e)^d \equiv M^{ed} \equiv M^{1 + k\phi(n)} \equiv M \pmod{n} \)

OAEP Implementation: Textbook RSA is deterministic. We implement OAEP to inject randomness via a Feistel network using Hash Functions (SHA-256/MGF1). This guarantees IND-CCA2 semantic security, protecting against padding oracles.

export const rsaEncrypt = (text: string, publicKeyPem: string): string => {
  const publicKey = forge.pki.publicKeyFromPem(publicKeyPem);
  // encrypt using RSA-OAEP with SHA-256 for padding randomness
  const encrypted = publicKey.encrypt(text, 'RSA-OAEP', {
    md: forge.md.sha256.create(),
    mgf1: { md: forge.md.sha1.create() }
  });
  return forge.util.encode64(encrypted);
};

6. Post-Quantum Cryptography (Quantum Mode)

6.1. Crystals-Kyber (Module-LWE)

To achieve quantum-resistance, the application implements Kyber768 (NIST FIPS 203 ML-KEM). It is based on the Module Learning With Errors (MLWE) problem.

All mathematics in Kyber occur over a specific polynomial ring:

\( R_q = \mathbb{Z}_q[X] / (X^{256} + 1) \)

Where \(q = 3329\). For a public matrix \( A \in R_q^{k \times k} \), a secret vector \( s \) and error vector \( e \) are sampled from a binomial distribution. The public key is \( (A, t) \) where \( t = As + e \).

KEM Decapsulation Math: When a message \(M\) is encapsulated, the receiver calculates:

\( v - s^T u = \lfloor \frac{q}{2} \rceil M + (e^T r + e_2 - s^T e_1) \)

Because the noise vectors (\(e, r, s, e_1, e_2\)) are incredibly small relative to \(q/2\), the receiver simply rounds the result, mathematically scrubbing away the noise to securely recover the exact shared AES key.

// Encapsulates a shared AES secret over the untrusted Socket.io network
export const kyberEncapsulate = (publicKeyBase64: string): { ciphertext: string, sharedSecret: string } => {
  const pk = new Uint8Array(forge.util.decode64(publicKeyBase64).split('').map(c => c.charCodeAt(0)));
  const c_ss = kyber.Encrypt768(pk); 
  return {
    ciphertext: forge.util.encode64(String.fromCharCode(...c_ss[0])), // Sent over network
    sharedSecret: forge.util.encode64(String.fromCharCode(...c_ss[1])) // Used locally as AES Key
  };
};
